## Overview

SolarGPT is a domain-specific AI application built for clean energy analysis — covering solar potential, grid economics, and power plant emissions for every U.S. county, state, and grid balancing authority. The platform ingests data from EIA-923, EIA-861, EIA-930, NREL Cambium, and Google Sunroof, and presents them through a fast, interactive UI built for energy analysts, site selectors, and policy researchers.

## Key Features

- **Geographic Profiles**: Every county, state, and grid region has a dedicated page with switchable profile tabs — Solar, Electricity Generation, Grid Economics, and Emissions — each backed by a different EIA data pipeline.
- **Power Plant & Utility Pages**: Plant-level generation and emissions (CO₂, SO₂, NOₓ) from EIA-923, linked to utility ownership via EIA-860. 885M metric tons of CO₂ tracked across 567 utilities.
- **Grid Intelligence**: Balancing authority network maps, interconnection queue data (97,736 rows, 64 NYISO snapshots from 2021–2026), and grid region subregion drill-downs.
- **Cambium Economics**: Long-run marginal cost, LCOE, and CO₂ intensity projections by scenario (Mid-case, High RE, High NG) rendered as 24-hour × 12-month cost heatmaps and area charts.
- **AI Chat**: Context-aware energy assistant powered by GPT-4o-mini, with awareness of the active geographic page and profile data.

## Technologies Used

- **Next.js 15 (App Router)**: Server components, dynamic routes, and shared `GeoDetailPage` layout used across county, state, GEA, plant, and utility pages.
- **Neon Postgres**: Serverless Postgres with 20+ tables and materialized KPI views covering solar, grid, generation, consumption, and emissions at every geographic level.
- **EIA Data Pipelines**: Python ingest scripts for EIA-923 (plant generation + emissions), EIA-861 (utility sales + service territory), EIA-930 (real-time grid flows), and NREL Cambium projections.
- **Google Maps + Solar API**: Rooftop solar potential at city and ZIP level, district boundary overlays.
- **Recharts**: Cost heatmaps, generation area charts, emissions line charts, and scenario comparison views.

## Challenges and Learnings

The biggest challenge was joining disparate government data sources that use inconsistent identifiers. EIA-923 plant IDs, EIA-861 utility numbers, EIA-860 plant-utility linkages, and Cambium grid region codes all use different schemas — reconciling them into a single Neon schema required careful foreign key design and several intermediate mapping tables.

The `GeoDetailPage` shared layout — which renders switchable profile tabs across wildly different data shapes (county vs. state vs. GEA vs. plant) — required a flexible `extraProfiles` prop pattern that is now the canonical template for all profile-driven pages in the portfolio.

## Outcome

SolarGPT covers every U.S. county (2,235 with generation data), all 50 states, and 20+ grid balancing authorities. It serves as both a live energy intelligence product and the reference architecture for domain-specific AI applications that combine multiple government data pipelines into a single, navigable interface.
