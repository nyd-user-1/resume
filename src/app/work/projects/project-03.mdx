## Overview

NSRgpt is a domain-specific AI application built for nuclear physicists and researchers. It wraps the Nuclear Science References (NSR) database — maintained by the National Nuclear Data Center at Brookhaven National Laboratory — in a modern semantic search and AI chat interface. The platform makes 88,000+ peer-reviewed nuclear science records discoverable through natural language, nuclide/reaction filters, and vector similarity search.

## Key Features

- **Semantic Search**: Queries are embedded via OpenAI's `text-embedding-3-small` model (256 dimensions) and matched against the full NSR corpus using a hybrid pipeline — pgvector semantic reranking on top of a PostgreSQL full-text pre-filter. 30-minute embedding cache reduces latency.
- **AI Chat (RAG Pipeline)**: GPT-4o answers questions grounded in real NSR records. The pipeline extracts nuclides, reactions, and author names from the query, runs parallel semantic + structured + author searches, fetches corroborating Semantic Scholar papers, and streams a cited response via SSE.
- **Record Browser**: Full NSR record browser with nuclide autocomplete, reaction autocomplete, Z-range sliders, year filters, and paginated results (99 per page). Each record links to DOI, EXFOR cross-references, and Semantic Scholar enrichment.
- **ENDF Report Viewer**: Inline PDF viewer for 300+ ENDF technical reports with date and author filters.
- **Research Feed**: Real-time Supabase Realtime feed of search and chat activity, aggregated every 15 minutes into trending nuclides and high-impact papers.

## Technologies Used

- **React 19 + Vite**: Fast SPA with React Router, TanStack Query, and Radix UI primitives.
- **Supabase (Postgres + pgvector)**: Managed Postgres with 256-dim vector embeddings on 88,000 NSR records, plus Auth, Realtime, and 7 Deno Edge Functions.
- **OpenAI API**: `text-embedding-3-small` for embeddings; GPT-4o for chat with streaming SSE responses.
- **Semantic Scholar API**: Batch and single-record enrichment for citation counts, abstracts, open-access status, and TLDRs — linked to NSR records by DOI.
- **TailwindCSS 4 + Recharts**: Dark/light theme, faceted search stat widgets, year distribution bar charts.

## Challenges and Learnings

The hardest problem was relevance. Nuclear science queries are highly technical — nuclide notation (`²³⁸U`), reaction notation (`(n,γ)`), and author name disambiguation all require domain-aware preprocessing before embedding. The solution was a structured extraction step before retrieval: parse nuclides and reactions from the query first, then run targeted structured search in parallel with semantic search, and merge results before sending to GPT-4o.

Building the hybrid `search_nsr_hybrid` Postgres RPC — which combines full-text tsvector pre-filtering with pgvector cosine similarity reranking in a single round-trip — reduced latency from ~800ms to ~200ms for typical queries.

## Outcome

NSRgpt makes one of the most specialized scientific databases in the world accessible through natural language for the first time. It demonstrates that domain-specific AI applications can deliver genuine research value in highly technical fields when the retrieval architecture is designed around the domain's actual query patterns — not generic keyword search.
